CUDA-Native GEMM Kernel:
Tiling, Vectorization & Roofline Analysis

Engineered a high-performance GEMM kernel in CUDA C++ from scratch, progressively optimizing from a naive baseline through shared memory tiling, register blocking, and float4 vectorized memory access. Achieved over 80% of cuBLAS throughput on an NVIDIA A100-SXM4-40GB GPU, diagnosed with NVIDIA Nsight Compute roofline analysis.

Key engineering decisions

  • Shared memory tiling eliminates redundant global memory reads; each tile is loaded once and reused across all dot-product accumulations within the block.
  • Register blocking increases arithmetic intensity per warp, pushing the kernel from memory-bound toward compute-bound on the roofline.
  • Float4 vectorization coalesces HBM reads into 128-bit transactions, maximizing memory bus utilization on A100 HBM2e and lifting throughput from 7.35 → 7.60 TFLOPS.
  • Memory padding applied to shared memory arrays to resolve 50% bank conflict rate diagnosed by Nsight Compute.
  • Occupancy bottleneck (25% theoretical, 95 regs/thread) accepted as a deliberate optimization trade-off: register blocking delivers higher per-warp IPC than the latency hiding from increased occupancy would provide.

Optimization pipeline

  • Applied TensorRT INT8 Post-Training Quantization with a representative calibration dataset; benchmarked precision vs. accuracy trade-offs across FP32, FP16, and INT8 on both LLM and CNN model families.
  • Leveraged TensorRT's automatic layer fusion to collapse Conv-BN-ReLU sequences and eliminate intermediate tensor materialization, reducing memory bandwidth pressure and kernel launch overhead.
  • Developed custom C++/CUDA TensorRT plugins (IPluginV2 interface) to implement non-native activation functions unsupported by the TensorRT op registry — enabling full graph execution without PyTorch runtime fallback.
  • Achieved 15% additional end-to-end latency reduction on edge LLM benchmarks by optimizing plugin memory footprint and eliminating unnecessary host-device transfers at plugin boundaries.
  • Profiled inference latency across batch sizes and sequence lengths using Nsight Systems to identify kernel scheduling bottlenecks and CUDA stream underutilization.

Engineering notes

  • INT8 calibration requires representative input distributions; implemented a custom entropy-based calibrator to avoid accuracy degradation on long-tail inputs.
  • IPluginV2 constraints around workspace allocation and serialization required careful state management for stateful activation functions in attention-based LLM layers.
  • Layer fusion is non-trivial when dynamic shapes are involved — resolved by freezing input/output dimensions at engine build time and maintaining separate engines per deployment configuration.

FlashAttention CUDA Kernel:
Fused Attention with Tiling & Online Softmax

Implemented a fused FlashAttention kernel from first principles in CUDA C++, targeting the key bottleneck in transformer inference: O(N²) HBM memory traffic from naive attention. By combining online softmax with SRAM-resident tiling, the kernel eliminates intermediate materialization of the full N×N attention matrix, achieving over 2× throughput improvement on RTX 4090 for sequence lengths up to 4096.

Technical approach

  • Standard attention (QKᵀV) requires materializing an N×N matrix to HBM — at sequence length 4096 this saturates HBM bandwidth, making the kernel memory-bound regardless of arithmetic throughput.
  • Implemented tiling in CUDA with shared memory buffers for Q/K/V tiles and per-thread running max/sum accumulators for online softmax, enabling a single-pass fused kernel that never writes the full attention matrix to HBM.
  • Diagnosed HBM bandwidth saturation and SM occupancy bottlenecks via Nsight Compute; refactored tile dimensions to maximize SRAM reuse per SM and improve arithmetic intensity, shifting the roofline operating point from memory-bound to compute-bound.
  • Benchmarked against PyTorch naive attention across sequence lengths 512–4096 on RTX 4090; throughput advantage exceeds 2× and widens with sequence length as the O(N²) HBM penalty of naive attention grows.
  • Causal masking for decoder-only models integrated into the tiling loop by zeroing future-token contributions within each tile, adding minimal compute overhead.

Engineering insights

  • Tile size selection is constrained by shared memory capacity per SM; larger tiles improve arithmetic intensity but risk register spilling — profiled via Nsight to find the optimal configuration.
  • Online softmax requires tracking running max and sum per query token; careful numerical handling prevents overflow in the exponential computation without sacrificing performance.

ReceiptInbox —
Serverless Receipt Parser & Smart Alerts

Cloud-native web application automating receipt data extraction from photos, PDFs, and emails. Users upload receipts; the system parses merchant, date, and total via AWS Textract OCR, applies ML-based spend categorization, stores results in a hybrid database, and pushes anomaly alerts via SES. Backend lead: responsible for the full serverless infrastructure and API layer.

PythonFastAPIAWS LambdaSQSTextractPostgreSQLDynamoDBS3JWTAWS SAM

Architecture highlights

  • Serverless REST API built with FastAPI on AWS Lambda via API Gateway — scales to zero when idle, eliminating baseline compute cost.
  • Presigned S3 URL workflow: the API generates short-lived upload tokens so the React frontend writes receipt files directly to S3, bypassing the Lambda execution boundary and the 10MB API Gateway payload limit.
  • Amazon SQS decouples upload acknowledgement from ML processing — the API returns immediately; Textract OCR and Comprehend categorization run asynchronously in background Lambda workers.
  • Hybrid data model: PostgreSQL on EC2 for relational user and job data; DynamoDB for high-throughput receipt metadata and processing status lookups.
  • JWT-based stateless authentication implemented from scratch; PostgreSQL hardened via pg_hba.conf and Security Group rules.

Key contributions

  • Designed the SQS fan-out processing pipeline — OCR worker triggers parse-and-categorize worker via queue chaining; failures isolated via Dead Letter Queue with automatic retry.
  • Built anomaly detection rules engine (duplicate charge detection, threshold-based spend alerts) with Amazon SES email notification delivery.
  • Authored AWS SAM infrastructure-as-code templates for reproducible deployment of Lambda functions, API Gateway, SQS queues, and IAM roles.

NLP · Deep Learning · PyTorch

Intent Classification
for ATIS Airline Queries

Systematic investigation into 21-class NLP intent classification on the ATIS benchmark dataset. Built the full PyTorch training pipeline from data ingestion through multi-layer LSTM with GloVe transfer learning, with a hypothesis-driven experimental methodology progressing from logistic regression baseline to an optimized deep learning architecture.

PythonPyTorchGloVe 100dLSTMTF-IDFTransfer Learning

Experimental progression

  • Baseline: Logistic Regression + TF-IDF, Macro F1 ≈ 0.30 — class imbalance makes accuracy misleading, confirming the need for Macro F1 as primary metric and a sequence-aware deep learning approach.
  • Experiment 3a: Single-layer LSTM with randomly initialized embeddings — limited generalization due to small training set (~5,800 samples) and no prior semantic structure.
  • Experiment 3b (central experiment): GloVe 100d pre-trained embeddings improved disambiguation of domain-specific synonyms ("plane" vs "aircraft" vs "jet") by leveraging semantic structure from billions of web tokens.
  • Experiment 4: Multi-layer LSTM (2 layers) with systematic tuning of learning rate and dropout — targeted overfitting reduction and F1 improvement on semantically similar intent pairs (flight vs flight_time).

Engineering challenges solved

  • Diagnosed "3.2 Loss" training stall: loss fixed at ~3.20 with near-zero accuracy. Root cause was average-length padding truncating valid sequences. Fixed with robust padding — model converged immediately.
  • GloVe 100d dimension vs. 128-dimensional model mismatch — resolved by refactoring model initialization to accept configurable embedding dimensions.
  • GloVe dataset exceeded GitHub's 100MB file limit — managed via .gitignore exclusion and Git history rewriting.

Machine Learning Research · Publication · Imperial College London

MOR-GANs: Multi-Output Regression
with Generative Adversarial Networks

Co-authored research paper published in Applied Sciences (2022), proposing MOR-GAN — a novel application of Wasserstein GANs as a flexible multi-output regression method. WGANs learn the full underlying data distribution rather than mapping inputs to point estimates, making them inherently capable of modeling multi-modal responses that defeat standard Gaussian Process Regression.

PythonTensorFlowKerasWGANGPRImperial College London

WGAN architecture

Key results

The challenge: multi-modal distributions

  • Standard GPR maps a single input to a single-valued posterior mean — when the true distribution is multi-modal (e.g., an annulus), GPR averages the ring and predicts the empty center, failing entirely.
  • WGANs are generative: the generator produces both independent and dependent variables as outputs, learning the full joint distribution. Multi-modal regression becomes a natural capability.
  • A custom prediction algorithm uses back-propagation through the trained generator to constrain the independent variable to a desired value — enabling point prediction from a fundamentally generative model.

My contribution

  • Software implementation of WGAN Generator and Critic in Python / TensorFlow / Keras, including the convolutional MOR-GAN variant for multi-output tasks.
  • Comparative benchmarking of WGAN against AutoEncoder, RNN, and CNN on multi-dimensional regression tasks derived from air pollution sensor data (EPSRC INHALE project).
  • Statistical analysis using Kolmogorov-Smirnov and Mann-Whitney U tests to quantify distributional match between generated and real data.

Semiconductor Fabrication · Materials Science

High-k Sol-Gel HfO₂
Gate Dielectric Fabrication

Cleanroom investigation into a low-cost deposition process for HfO₂ High-k gate dielectrics as a replacement for thermally grown SiO₂ in scaled CMOS. HfO₂ is the industry-standard High-k material enabling continued gate capacitance scaling beyond the SiO₂ leakage limit; doped variants exhibit ferroelectric properties compatible with FeRAM and advanced logic. Conducted at NCTU Electrophysics Lab.

Sol-GelSpin CoatingHfO₂MOS CapacitorC-V / I-VNCTUCMOS

Fabrication process

  • Sol preparation via hydrolysis and condensation of HfOCl₂·8H₂O in ethanol; aging at room temperature for target viscosity and nanoparticle suspension stability.
  • Silicon substrate cleaning followed by HMDS hydrophobic surface treatment to minimize contact angle and ensure uniform sol spreading during spin-coating.
  • Spin-coating deposition of HfO₂ sol layer followed by soft bake at 150–200°C for solvent removal and initial film densification.
  • Full MOS-C (Metal-Oxide-Semiconductor Capacitor) test structure fabrication for electrical characterization.

Electrical characterization

  • C-V measurements to extract dielectric constant (k-value), flat-band voltage, and interface trap density; compared against thermally grown SiO₂ reference.
  • I-V measurements to quantify leakage current density — the primary motivation for High-k dielectrics is suppressing gate leakage at equivalent oxide thickness (EOT).
  • Sol-Gel as a low-cost alternative to ALD — demonstrated process feasibility for research-scale HfO₂ deposition without high-vacuum equipment.

Optics · Computational Physics · MATLAB

Optical Vortex Generation:
Astigmatic Transformation of Hermite-Gaussian Beams

Mathematical and computational analysis of structured light fields at NCTU Electrophysics Lab. Core objective: convert HG laser modes into Laguerre-Gaussian modes carrying Orbital Angular Momentum via an astigmatic cylindrical lens converter, then visualize the resulting 3D wave patterns and optical vortex structures in MATLAB.

MATLABHermite-GaussianLaguerre-GaussianOAMPhase SingularityNCTU

Physics background

  • HG modes are described in Cartesian coordinates; LG modes are the cylindrical counterparts, carrying well-defined OAM of lℏ per photon.
  • An Astigmatic Mode Converter (two cylindrical lenses) introduces a relative π/2 Gouy phase shift between orthogonal mode components, mapping HG(n,m) → LG(p,l).
  • The resulting LG beam has a phase singularity (optical vortex) at the beam axis — a point of zero intensity where the helical wavefront phase is undefined. Topological charge l determines vortex handedness and multiplicity.

Computational work

  • Modeled full HG mode propagation via the paraxial wave equation, computing beam waist evolution w(z) and Rayleigh range z_R from fundamental Gaussian parameters.
  • Implemented astigmatic transformation in MATLAB to synthesize LG mode intensity and phase distributions, confirming donut-shaped intensity profile and helical phase structure.
  • Visualized 3D particle trajectories in the optical potential well generated by the OAM-carrying LG beam, validating theoretical models of optical trapping and micro-particle manipulation via OAM transfer.
